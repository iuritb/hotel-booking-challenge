# Teste prãtico de desenvolvimento front-end com nuxt 

## Objetivo do Teste 

    Desenvolver uma pequena aplica~c"ao usando nuxt 3 para simular parte do sistema
  de reserva de hoteis. O foco serã implementar uma interface funcional que permita
  pesquisar, comparar e reservar hoteis. 

## Requisito do projeto 
  1. Pesquisa de hotéis: 
      Criar uma interface que permita ao usuários inserir critérios de pesquisa 
    como destino datas de check-in e check-out, número de quartos e numero de
    hóspedes.
      Exibir os resultados da pesquisa em uma lista que pode ser ordenada por 
    preço, avaliações e outros critérios relevantes.

  2. Comparação de Opções:
    Permitir que o usuário selecione multiplos hotéis dos resultados de busca 
    para comparar suas caracteristicas lado a lado.

  3. Reserva de Quartos:
    Implementar um formulário para reservar um quarto em um hotel selecionado, 
    solicitando informacões como nome do usuário, contato e detalhes de pagamento.

  4. Notificações:
      Implementar um mecanismo de notificacão que informe ao usuário sobre o 
    status de sua reserva.

## Critérios Técnicos Especificos: 
  - Utilizar Nuxt3 juntamente com CompositionAPI
  - Utilizar Typescript
  - Utilizar Pinia para Auth do usuário.
  - Utilizar Tailwind para estilizaçãodas páginas
  - UtilizarNuxtLayers
  - Implementar chamadas de API simuladas (mock) via SSR para busca e reserva de de hotéis.
  - Assegurar que a aplicação seja responsiva e acessível.
  - Otimizar o desempenho da aplicação (LazyLoading, Codesplitting).
  - Configurar um pipeline básicode CI/CD para o projeto usando ferrmentas como github actions

## Tarefas
  1.SetupdoProjeto:
    Configurar um novo projeto Nuxt usando SSR.
    Estruturar o projeto com componentes reutilizáveis e seguir as melhores práticas do Nuxt como DirectoryStructure.

  2.Implementação de funcionalidades: 
    Desenvolver componentes conforme os requisitos listados.
    Integrar Composition API conforme necessário para a lógica da aplicação.
    Criar os endpoints via SSR do Nuxt.

  3.Testes:
    Escrever testes básicos para os componentes utilizando Vitest ou outra
    ferramenta de teste de sua escolha.

  4.Documentação:
    Documentar o projeto e o código, explicando as escolhas de design earquitetura.


## Entrega:
  - O código deve ser subbetido através de um repositório Git (por exemplo,GitHub, GitLab).
  - Incluir instruções de instalação e execução no README do projeto.
  - Critérios de Avaliação Completo cumprimento dos requisitos funcionais.
  - Qualidade e organização do código.
  - Eficiência dassoluções implementadas.
  - Documentação clara e detalhada.
