## Descrição do Desafio:

Desenvolver uma pequena aplicação Nuxt 3 que simule um sistema de reserva de hotéis. O objetivo é criar uma interface funcional para pesquisar, comparar e reservar hotéis, focando nos critérios técnicos especificados.
  ____________________________________________________________________________________________________________________________
## Requisitos Detalhados:

  * *Funcionalidades Principais:*
      - Pesquisa de Hotéis: Criar uma interface para que usuários insiram critérios de pesquisa (destino, datas de check-in/check-out, número de quartos e número de hóspedes).
      - Exibição e Ordenação de Resultados: Exibir os resultados da pesquisa em uma lista que pode ser ordenada por preço, avaliações e outros critérios relevantes.
      - Comparação de Opções: Permitir a seleção de múltiplos hotéis para comparação lado a lado de suas características.
      - Reserva de Quartos: Implementar um formulário para reservar um quarto em um hotel selecionado, solicitando informações do usuário (nome, contato, detalhes de pagamento).
      - Notificações: Implementar um mecanismo de notificação para informar o usuário sobre o status de sua reserva.

  * *Tecnologias/Stack Obrigatórias:*
      - Nuxt 3 (juntamente com Composition API)
      - TypeScript
      - Pinia para autenticação do usuário (Auth)
      - Tailwind CSS para estilização das páginas
      - Nuxt Layers

  * *Integração com API:*
      - Implementar chamadas de API simuladas (mock) via SSR do Nuxt para busca e reserva de hotéis.

  * *Design/Experiência do Usuário*
     - Assegurar que a aplicação seja responsiva e acessível.

  * *Outras Restrições/Considerações:*
    - Otimizar o desempenho da aplicação (LazyLoading, Code Splitting).
    - Configurar um pipeline básico de CI/CD para o projeto usando ferramentas como GitHub Actions.
    - Estruturar o projeto com componentes reutilizáveis e seguir as melhores práticas do Nuxt (Directory Structure).
    - Escrever testes básicos para os componentes (usando Vitest ou ferramenta similar).
    - Documentar o projeto e o código, explicando as escolhas de design e arquitetura.
    - O código deve ser submetido via repositório Git (GitHub/GitLab) com instruções de instalação e execução no README.
